# StayHub 酒店管理系统 —— 项目说明文档

> Java 大一下期末作业  |  Swing GUI + CLI 双界面  |  纯 Java SE，零外部依赖

---

## 一、项目概述

StayHub 是一套**酒店前台管理桌面软件**，支持：
- 前台接待：办理入住、退房、查看房态
- 经营管理：营业统计、订单查询、员工管理
- 客房服务：清洁状态追踪

**两个入口**共享同一套业务逻辑：
| 入口 | 文件 | 说明 |
|------|------|------|
| 图形界面 | `StayHubUI.java` | Swing GUI，全功能，推荐使用 |
| 命令行 | `Main.java` | 控制台交互，功能对等 |

---

## 二、项目架构

```
Main.java (CLI)  /  StayHubUI.java (Swing GUI)
         \               /
          HotelManager.java          ← 全部业务逻辑
         /        |        \
   Room.java  Customer.java  Order.java  Staff.java  User.java   ← 数据模型
         \        |        /
          FileUtil.java              ← 文件持久化
```

### 分层说明

| 层 | 职责 | 文件 |
|----|------|------|
| **UI 层** | 用户交互、表单验证、数据显示 | `StayHubUI.java`, `Main.java` |
| **业务层** | 入住退房流程、状态流转、数据校验 | `HotelManager.java` |
| **模型层** | 数据结构定义、状态常量 | `Room.java`, `Order.java`, `Customer.java`, `Staff.java`, `User.java` |
| **持久层** | 读写磁盘、文件容错 | `FileUtil.java` |

### 数据存储

- 使用 Java 原生**对象序列化**（`Serializable`）
- 数据保存在项目根目录 `.dat` 文件中
- **写入策略**：先写 `.tmp` 临时文件，成功后再替换原文件，防止写入中途崩溃导致数据损坏
- 文件不存在或损坏时自动返回空列表，首次启动自动创建管理员账号 `admin / 123456`

## 三、功能清单与实现方法

### 1. 用户认证

| 功能 | 实现位置 | 方法 |
|------|----------|------|
| 登录验证 | `HotelManager.login()` | 遍历 `users` 列表，调用 `User.login()` 比对用户名密码 |
| 注册账号 | `HotelManager.register()` | 校验用户名 ≥3 位、密码 ≥4 位、用户名唯一；返回 `null` 成功/错误信息 |
| 默认管理员 | `HotelManager()` 构造函数 | 首次启动 `users` 为空时自动创建 `admin / 123456` |

### 2. 房间管理

| 功能 | 实现位置 | 方法 |
|------|----------|------|
| 添加房间 | `HotelManager.addRoom()` | 校验价格 > 0、房号不重复；新房间默认"空闲" |
| 查看房态 | `StayHubUI.refreshRoomDisplay()` | HTML 表格渲染，彩色显示：<span style="color:green">绿色=空闲</span>、<span style="color:blue">蓝色=已入住</span>、<span style="color:orange">橙色=待清洗</span> |
| 房间状态常量 | `Room.java` | `STATUS_FREE`（空闲）、`STATUS_OCCUPIED`（已入住）、`STATUS_CLEANING`（待清洗）、`STATUS_MAINTENANCE`（维修中） |

### 3. 入住登记

| 步骤 | 实现 |
|------|------|
| 校验入住天数 | `days > 0`，否则返回错误信息 |
| 查找空闲房间 | 遍历 `rooms`，匹配房号 + 状态为 `STATUS_FREE` |
| 创建客户记录 | 添加到 `customers` 列表 |
| 更新房间状态 | `room.setStatus(STATUS_OCCUPIED)` |
| 生成订单 | 订单号 = `"ORD" + 时间戳`，状态 = `STATUS_CHECKED_IN` |
| 满7天优惠 | `if (days >= 7) totalAmount *= 0.9` 自动打9折 |
| 持久化 | 同时保存 `customers.dat`、`rooms.dat`、`orders.dat` |

### 4. 退房结算

| 步骤 | 实现 |
|------|------|
| 输入订单号 | 弹出输入框 |
| 查询订单 | `HotelManager.getOrderById()` — 遍历 `orders` 列表按 ID 匹配 |
| 展示账单 | 弹出确认框，显示：订单号、客户名、身份证、房号、入住日、天数、单价、**总计**、折扣信息 |
| 确认退房 | 用户点击确认后执行 |
| 更新订单状态 | `order.setStatus(STATUS_CHECKED_OUT)` |
| 房间变待清洗 | `room.setStatus(STATUS_CLEANING)` — 退房后需清洁才能再入住 |
| 随机趣味提示 | 5条退房祝福语中随机选一条显示 |

### 5. 清洁管理

| 功能 | 实现 |
|------|------|
| 手动标记待清洗 | `HotelManager.markRoomForCleaning()` — 仅"空闲"房间可标记 |
| 标记已清洁 | `HotelManager.markRoomCleaned()` — 仅"待清洗"房间可恢复"空闲" |
| 清洁状态列表 | `StayHubUI.showCleaningDialog()` — 弹窗列出所有房间，显示彩色状态标签 + 操作按钮 |

### 6. 订单管理

| 功能 | 实现 |
|------|------|
| 查看全部订单 | `StayHubUI.refreshOrderDisplay()` — HTML 表格，含订单号、客户、房间、日期、天数、金额、状态 |
| 搜索订单 | `HotelManager.searchOrders(keyword)` — 不区分大小写匹配客户姓名、房间号、订单号 |
| 订单状态常量 | `Order.java`: `STATUS_CHECKED_IN`（入住中）、`STATUS_CHECKED_OUT`（已退房） |

### 7. 营业统计

| 统计项 | 数据来源 |
|--------|----------|
| 房间概况 | 遍历 `rooms`：总数、空闲、已入住、待清洗、维修中 |
| 入住率 | `已入住数 ÷ 总房间数 × 100%`，带颜色预警（≥80%红、50-80%橙、<50%绿） |
| 在住营收 | 遍历 `orders`，过滤"入住中"状态，累加 `totalAmount` |
| 累计营收 | 遍历所有 `orders` 累加 `totalAmount` |
| 累计客户 | `customers.size()` |
| 实现 | `HotelManager.getStatistics()` 组装格式化文本 |

### 8. 员工管理

| 功能 | 实现 |
|------|------|
| 添加员工 | `HotelManager.addStaff()` |
| 编辑员工 | `HotelManager.updateStaff()` — 按工号查找，更新职位和薪资 |
| 删除员工 | `HotelManager.removeStaff()` — 按工号查找并移除 |
| UI | `StayHubUI.showStaffDialog()` — 弹窗列表，每行有编辑/删除按钮 |

## 四、UI/UX 设计亮点

### 视觉设计

| 设计点 | 实现方式 |
|--------|----------|
| HTML 彩色表格 | `JEditorPane` 设置 `contentType = "text/html"`，通过 CSS class 控制颜色 |
| 暖金色点缀 | `COLOR_GOLD = #B9966A`，标题下划线、分隔线使用 |
| 暖白底色 | `COLOR_DISPLAY_BG = #FCFBF8`，替代纯白，纸张质感 |
| 登录页品牌感 | 大字 STAYHUB + 金色副标题 + 分隔线，次要操作用灰色按钮 |
| 侧栏深蓝灰 | `COLOR_MENU_BG = #263240`，相比纯黑更柔和高级 |

### 物理动效

| 动效 | 实现方式 |
|------|----------|
| 按钮按压回弹 | `MouseListener`：按下时 `setBorder` 上移下边距 + `darken()` 压暗颜色；松开后 `Timer` 分两阶段弹跳归位（40ms 放大 → 90ms 恢复） |
| 菜单悬停浮起 | `mouseEntered` 时将边框微调 1px 放大 + 变蓝，`mouseExited` 恢复 |
| 状态栏呼吸动画 | `javax.swing.Timer` 每 500ms 循环切换"系统就绪 / 系统就绪. / 系统就绪.. / 系统就绪..." |
| 操作成功绿闪 | `triggerSuccessFlash()`：`Timer` 每 40ms 一帧，8 帧内从淡绿渐变回底色 |
| Tooltip 悬停提示 | 每个菜单按钮 `.setToolTipText()` |
| 关闭确认 | `WindowListener` 拦截关闭事件，弹出确认框防误触 |

## 五、技术要点

### 设计模式

| 模式 | 应用 |
|------|------|
| **单一职责** | `Room`/`Order`/`Customer` 各自只管自己的数据；`HotelManager` 只管业务逻辑；`FileUtil` 只管文件读写 |
| **分层架构** | UI → 业务逻辑 → 数据模型 → 文件持久化，每层只与相邻层交互 |
| **常量替代硬编码** | `Room.STATUS_FREE` 替代字符串 `"空闲"`，编译期检查、IDE 可追溯 |

### 数据安全

- 每次数据修改后**立即写回磁盘**
- 写入采用 `.tmp` 临时文件 + 替换策略，防止写入中途崩溃
- 文件损坏时返回空列表，不抛异常

### 错误处理

- 业务方法统一返回 `String`：`null` = 成功，非 `null` = 错误信息
- UI 层根据返回值决定弹成功提示还是错误提示

## 七、答辩准备建议

**老师可能问的问题：**

1. **为什么选择 Swing 而不是 JavaFX / Web？**
   > Swing 是 Java SE 标准库，无需额外依赖，适合课程作业的快速交付。

2. **数据存在哪里？重启会丢吗？**
   > 使用 Java 原生序列化存储为 `.dat` 文件，每次操作后立即写盘，重启不丢数据。

3. **如果程序崩溃数据会损坏吗？**
   > `FileUtil` 采用先写 `.tmp` 再替换的策略，最坏情况下只丢失最后一次操作，原有数据完整。

4. **为什么退房后房间变成"待清洗"而不是直接"空闲"？**
   > 模拟真实酒店流程：客人退房 → 保洁清洁 → 房间恢复可售。这是从"交作业"到"做产品"的思维。

5. **入住满7天9折是怎么实现的？**
   > `HotelManager.checkIn()` 中一行 `if (days >= 7) totalAmount *= 0.9;`，简单但体现了业务规则设计能力。

**展示流程建议：**
1. 先展示登录页 → 注册 → 登录
2. 录入 3-4 个房间（不同类型和价格）
3. 办理入住（选一个满 7 天的，展示折扣）
4. 查看房态（入住率变化）
5. 办理退房（展示账单弹窗）
6. 查看清洁管理（房间自动变"待清洗"）
7. 标记已清洁 → 房间恢复空闲
8. 展示营业统计面板
9. 搜索订单
10. 关掉窗口（展示退出确认）
